Bitcore Wallet Service

Bitcore Wallet Service

A Multisig HD Wallet Service Software, with minimun server trust.

Motivation

After 6 months working on Copay, it become obvious a new architecture was needed to handle our use cases. 

        


        


Motivation



· Simpler networking

· Simpler client software

· Simpler to support

· No private keys server-side 

    (Device based authentication)


        




        


Data Storage





Clients: WCD (Wallet Critical Data)



 · Extended Private Key
 · Copayer's Extended Public Keys


        


        


Data Storage





Server: WED (Wallet Extended Data)



 · Pending Transaction Proposals 
 · Transaction Proposal History
 · Addresses

Sensitive TX fields are encryted and 
unknown to the server

Security


At the client (using the official lib):


   All responses from the server are checked.

   ✔ Address are derived by each client
   ✔ Transaction Proposals are signed by clients 
        and validated by clients
   ✔ Copayer's xpub are signed by wallet's join secret 
        and validated by clients


        


        


Security



At the server:

 ✔ All requests to server are signed by the copayers
     and checked by the server (similar to bitauth)
 ✔ There are 2 signing keys: one for read-only operation
     and one for read-write operations


        


        


What if?



 The server disappear

   ✔ Funds are not compromised

   ✘ Pending transaction proposals are lost
   ✘ Some metadata on historical transaction is lost

❱ With the required quorum the wallet can be
  easily recreated on other server, and funds accessed.



        


        


What if?



  The server is comprommised 

   ✔ Funds are not compromised
   ✔ Transaction proposals or address responses
       can not be tampered.
   ✔ No linkage to user identity is possible

   ✘ Wallet usage can be denied
   ✘ Tx Rejections can be faked
   ✘ Transaction history and the
       and extended public keys are compromised

❱ With the required quorum the wallet can be
  easily recreated on other server, and funds accessed.



        


        

Goodies: Installation

    ✔ Extremely simple to install and run

    ✔  Based on Bitcore > 0.8
        






Goodies: Concurrency

    ✔ Concurrent access (among copayers)
    ✔ Concurrent access among same copayer
    ✔ No TXPs / UTXOS sync problems (!)
    ✔ Async wallet creation


        





Goodies: Access levels


Copayers can 'clone' themselves, with a different access level
to give access to certain software:

    'readonly' 
            (e.g for a 'balance notificator')
    'readwrite' 
            (e.g. for a 'consolidator proposer' or a 'hot-wallet limiter')
    'full' 
            (e.g. for a signing agent with limits)

    And this just means chopping some fields of the WCD!






Goodies: Air Gapped Device Support WIP!


Keys can be generated offline, and tx proposals
exported to that device to be signed.

    $ bit genkey
    $ bit export --access readonly -o toproxy.dat

    $ bit import toproxy.dat
    $ bit join [secret]
